Q1. What is object-oriented programming?
Ans. In object oriented programming, the data required for the program is divided and stored into restricted, independent and discrete area of computer memory called object which can be accessed directly, only by certain specific procedures defined in class to which the object belong to. The data of an object cannot be accessed directly by procedures of other class’ objects. However, the data of one object can be accessed indirectly by invoking or calling its procedures by procedures of other class’ object. To make an application run, objects of different classes call or invoke one another’s procedures.


Q2. What do you mean by sending or passing messages?
Ans. The objects of different classes collaborate and interact by sending or passing messages to one another. To make an application run, objects of different classes call or invoke one another’s procedures.


Q3. What are the four OOP principles? Explain each of them.
Ans. The four OOP principles are encapsulation, abstraction, inheritance and polymorphism.
Encapsulation
Encapsulation is the characteristic that data in an object can be accessed only by the method or code that is defined in its class. This mechanism keeps both data and code safe from outside interference and misuse. This is achieved by bundling data and code in the class.
Abstraction
Abstraction is a concept of hiding unnecessary details and representing only the very essential features. Abstraction permits the programmer to look at something without being concerned with the internal details.
Inheritance
Inheritance is the process of creating new class, called subclass (also called derived class, extended class or child class), from existing class called the superclass (also called base class or parent class). The subclass inherits fields and methods of the superclass but can add new fields and methods of its own.
Polymorphism
The ability of an object to respond, each in its own way, to identical methods calls is called polymorphism. Method overloading and method overriding are two of the ways that Java implements polymorphism. This can only happen when there is parent and child relationship between classes using inheritance.

Q14. How do you create or allocate an object of a class?
Ans. The new operator dynamically allocates (that is, allocates at run time) memory for an object and returns the reference or binary address of the object. It has this general form:
classname ref-var = new classname( );
Here ref-var is a reference variable, which stores the address of the newly created object. The classname is the name of the class that is being instantiated.


Q15. What are instance variables?
Ans. Instance variables are declared in the body of the class but outside the methods. When an object or instance of a class is created, memory is allocated for the instance variables declared in the class.


Q16.What is a method? Give advantages of using method.
Ans. A method is group of instructions or a block of code that has a name. Methods define the object’s behaviour, that is, the various operations that an object can perform. Methods are used due to reasons given below.
1. Methods help us to break a complex task in a program into smaller manageable parts. This modular approach makes a program relatively simple and easier to maintain.
2. Method is reusable, that is, once designed, can be invoked and executed any number of times. This feature helps us to reduce the size of the source code or program.
3. Lastly, the method hides from its user the complex task that it performs.


Q17. What are the elements or components of method declaration?
Ans. The essential required elements of a method declaration are given below.
1. Method name.
2. Method name is followed by empty pair of parentheses, (), or a comma-delimited list of parameters, preceded by their data types, enclosed in parentheses.
3. Method name is preceded by return type i.e. the data type of the value returned by the method or voidif the method does not return a value.
4. The method body which encloses the method’s code, including the declaration of local variables, between braces, {}.
Apart from this, other components of method declaration may be modifiers such as public, private etc. and an exception list. Here is an example of a typical method declaration:
public long calc(int a, float b) throws IOException
{
//method body
}


Q18. What is the purpose of return statement?

or return data or object reference;


Q19. What is the difference between arguments and parameters?
Ans. Arguments are the actual values viz. data or object references that are passed in by the calling method, to the called method, when it is invoked. Parameters refer to the list of variables in the method declaration that receive the values viz. data or object references that are passed in by the calling method, to the called method, when it is invoked.


Q20. How are primitive and reference data type arguments passed into methods? Explain.
Ans. Passing Primitive Data Type Arguments
Primitive type value is passed to a method by value. This means that when a method is called the primitive type value of the argument of the calling method is copied into the parameter of the called method. The called method works with the copy rather than with the original data. Therefore, changes made to the value of the parameter will have no effect on the value of the argument. When the method returns, the memory allocated for the parameter is freed but the argument still contains the original data.
Passing Reference Data Type Arguments
Reference type value is also passed to a method by value. This means that when a method is called the reference type value of the argument of the calling method i.e. the object reference or address is copied into the parameter of the called method. Now the address stored in the parameter also refers to the original object and thus the parameter can access the original data in the object and may change it if the object is mutable.
However, if address of the object, stored in the parameter is changed, the address of the object, stored in the argument will not be changed. When the method returns, the memory allocated for the parameter is freed but the argument still contains the original object reference.


Q21. How are primitive type and reference type arguments passed to a method?
Ans. Both primitive type and reference type arguments are passed to a method by value.


Q22. What is the difference between pure and impure method?
Ans. The difference between pure and impure method is given below.
 1. The only input to a pure method is the arguments passed to it and the only output is its return value.
2. A pure method has referential transparency, meaning it always returns the same output given the same input.
3. To achieve referential transparency, the pure method should not have any side effects. Side effects are the factors due to which the method gives a different output when invoked next time, for the same input.
(a) A pure method cannot use a static variable or modify it.
(b) It cannot use or modify the current object.
(c) The parameter of a pure method cannot be of reference type.
(d) A pure method cannot perform input/ output operations.
4. A pure method can use local variables and local objects and modify them if required during program execution. Similarly, a pure method can use parameters and modify it, provided the parameters are of primitive type and store the copy of the arguments.
5. A pure method can only invoke a pure method.

Impure methods do not support the above rules.

Q30. What do you mean by this reference? What is its purpose and where can it be used?
Ans. In Java, this is an invisible reference that automatically points or refers to the object that invokes a member. In other words, the keyword this, is an invisible reference variable that stores the address of the current object or invoking object. Reference this may be used to access the fields (instance variables or static variables) and methods of the class, though its use is redundant as fields and methods of a class can be accessed directly through non-static or instance method of that class. Most importantly, this reference can only be use from within a non-static method.


Q31. What are primitive wrapper classes? Explain. Name the primitive wrapper classes.
Ans. Primitive data types such as int, double, char etc. are not objects. To create an object representation of these primitive data types Java provides classes, which encapsulate, or wrap, the data type within a class. These classes are commonly referred to as type-wrappers or primitive wrapper classes. Thus, each type-wrapper class enables you to manipulate primitive-type values as objects. Primitive wrapper classes are Byte, Short, Integer, Long, Float, Double, Character and Boolean.


Q32. What do you mean by static variable or class variable?
Ans. Static variables are declared in the body of the class but outside the methods. Memory for the static variables is allocated only once when the class is loaded in the memory. When objects of a class are created, memory is not allocated for the static variable. Instead, all the objects of the class share the same static variable.


Q33. What do you mean by static method or class method? Method declared as static have several restrictions. Give any two of them.
Ans. A static method performs a task that does not depend on the content of any object. A static method cannot access instance variables or instance methods directly but can access static variables and static methods. To invoke a static method from outside its class, class name is used instead of object reference. The general form is given below.
classname.method( );
Here, classname is the name of the class in which the static method is declared.


Q34. What is an array? Give the general form of one-dimensional array. How is a specific element in one-dimensional array accessed?
Ans. An array is a container object that holds a fixed number of values (data or object references) of a single type. Array is a reference type. The general form of a one-dimensional array is
type [ ] array-var = new type [size];
Here name of the array, array-var is a reference variable that refers to an object or contains binary address of an object that holds fixed number of values of a single type.
A specific element in one–dimensional array is accessed by its index or subscript. For example, if a is an array of integers, then a[i] is the ith integer in the array.


Q35. Give the general form of two-dimensional array. How is a specific element in two- dimensional array accessed?
Ans. The general form of a two-dimensional array is
type [ ] [ ] array-var = new type [row size] [column size];
A specific element in a two-dimensional array is accessed by its row index and column index. For example, if a is a two-dimensional array of integers, then a[i][j] is the integer present in ith row and jth column in the array.


Q36. What is an index or subscript of a string?
Ans. An index is a unique number given to each character of a string. The number given to the first character of a string is zero. The number given to the second character of a string is one and so on.


Q37. What is the difference between linear search and binary search?
Ans. Technique of binary search can only be implemented on an array that is arranged in ascending or descending order. However, to implement linear search it is not essential that the array should be sorted.
Binary search is more efficient and fast searching algorithm than linear search because each comparison eliminates from consideration half of the elements in the array. Linear search algorithm is slow as it searches each element in an array sequentially until it finds the correct element.

Q41. What is the difference between public and private access specifier?
Ans. The public members of a class can be invoked by the code of any other class. On the contrary, the private members of a class can be invoked only by the code in the same class. Therefore, any other code that is not a member of the class cannot invoke private members.

Q43. How can you add a class into a package? Give its general form. What are the essential requirements or prerequisites to add a class into a package?
Ans. The package statement is used to place classes inside a package. You must put the name of the package at the top of the source file, before the code that defines the classes in the package. The general form of the package statement is:
package pkg;
Here, pkg is the name of the package. For example, the following statement creates a package called corejava.
package corejava;
You must place files in a package into a directory or folder that matches the package name. For example, all class files in the package corejava must be stored in a folder corejava.


Q44. How can you use a package or refer to a class in the package? Give its general form.
Ans. All of the standard classes of Java are stored in some named package. Java’s import statement is used to refer to the class in the package. These classes are imported because the program uses the objects and methods of these classes. The general form of the import statement is:
import pkg1.pkg2.classname;
or import pkg1.pkg2.*;
Here, pkg1 is the name of package, and pkg2 is the name of sub-package inside the outer package separated by a dot (.). You can specify either an explicit classname or a (*), which indicates that the Java compiler should import the entire package or in other words all the classes stored in that package.
This code fragment shows both forms in use:
import java.lang.Math;
import java.io.*;


Q45. What is an exception? What do you mean by exception handling?
Ans. An exception is an event or error, which occurs during the execution of a program, which disrupts the normal flow of the program’s instructions. The procedure adopted to deal with these exceptions to prevent the program from crashing or sudden termination at run time is called exception handling.


Q46. How will you handle an exception? What is the purpose of finally block? Give their general form.
Ans. To handle an exception, the code to be monitored, is enclosed inside a try block. Immediately following the try block, includes a catch clause that specifies the exception type that you wish to catch.
Once an exception is thrown, program control transfers out of the try block into the catch block. Once the catch statement has executed, program control continues with the next line in the program following the entire try/ catch mechanism.
The finally block always executes when the try block exits. The finally block will execute whether or not an exception is thrown. The finally clause is optional. The general form of try/ catch/ finally block is:
try
{
//block of code to monitor for errors
}
catch(ExceptionType1 exOb)
{
//exception handler for ExeptionType1
}
catch(ExceptionType2 exOb)
{
//exception handler for ExeptionType2
}
………
finally
{
//block of code will execute whether or not an exception is thrown
}

(ii) ArrayIndexOutOfBoundsException – This error occurs if a program attempts to access an element at a non-existent index in an array.

(b) StringIndexOutOfBoundsException

import java.lang.Math;

Q13. A primitive wrapper class wraps primitive data type in a class. Explain its purpose.
Ans. Primitive data types such as int, double, char etc. are not objects. Q18. Explain briefly the binary search technique.
Ans. Technique of binary search is implemented on an array already arranged in ascending or descending order. Assume that the array is arranged in ascending order. Item being searched for is compared with the item at the middle of the array. If they are equal, the search has been completed successfully. If the item to be searched for, is less than the middle element, the search process is repeated in the first half of the array, otherwise, the process is repeated in the second half. Thus each, iteration of binary search tests the middle value of the remaining array and, if the element is not found, eliminates half of the remaining array.
